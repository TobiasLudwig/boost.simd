Vectorized Struct Reference
Concepts » Types

Description

A type is Vectorized if it represents a type which operations are performed in accordance with the SIMD execution model.

Requirements

Vectorized types satisfy DefaultConstructible, CopyConstructible, CopyAssignable and MoveAssignable. Lvalues of Vectorized types satisfy Swappable.

Vectorized does not refine EqualityComparable (and hence Regular) because operator== returns Vectorizable instead of bool.

For any Vectorized type V, the following expressions are valid:

Expression Type Notes
V::storage_type native storage type
V::value_type T Satisfies Vectorizable
V::reference proxy reference to T Convertible to T
V::const_reference proxy reference to T const Convertible to T const
V::iterator iterator pointing to T RandomAccessIterator convertible to const_iterator
V::const_iterator const iterator pointing to T
V::difference_type signed integer type
V::size_type unsigned integer type Must be able to represent all positive values of difference_type
V::alignment constant expression of unsigned integer type The value of the preferred alignment for V
V::static_size constant expression of unsigned integer type The number of Vectorizable elements stored in an instance of V
V::template rebind<U> W For any Vectorizable type U, W::template rebind<T> is V
V::template resize<N> W For any integral constant N, W::template resize<V::static_size> is V
V::template retype<U,N> W For any integral constant N and any Vectorizable type U, W::template retype<T,V::static_size>is V
