Trigonometric functions.
User API » Functions and Constants

Description

Algorithms for computing scalar and SIMD version of trigonometric and inverse trigonometric functions and some utilities.

All these functions can be included individually or all of them just by including <boost/simd/trigonometric.hpp>

Except a meaningful case (namely cospi, as \(\cos(\pi n) = (-1)^n\) for integral n) these functions are only provided for floating (scalar or SIMD) inputs.

  • Direct trigonometric functors

    name name name name name name
    cos cosd cospi cotd cot cotpi
    cscd csc cscpi secd sec secpi
    sin sinc sincos sincosd sincospisincpi
    sind sinpi tan tand tan tanpi

    'pi' (resp. 'd') suffix indicates that the input is to be done in \(\pi\) multiples (resp. degrees).

    sinc and sincpi are 2 versions of the sinus cardinal function (resp. \(\frac{\sin x}{x}\) and \(\frac{\sin \pi x}{\pi x}\)).

    sincos, sincosd and sincospi allow the simultaneous computation of sine and cosine at lower cost.

    sec and csc are the secant (inverse of the cosine) and cosecant (inverse of the sine)

    Example:
    #include <boost/simd/trigonometric.hpp>
    #include <boost/simd/pack.hpp>
    #include <boost/simd/function/enumerate.hpp>
    #include <boost/simd/constant/pi.hpp>
    #include <iostream>
    namespace bs = boost::simd;
    using pack_ft = bs::pack <float, 8>;
    int main()
    {
    pack_ft p_pi = bs::enumerate<pack_ft>(-1.0f, 0.25f);
    pack_ft p_ra = p_pi*bs::Pi<pack_ft>();
    pack_ft p_dg = bs::indeg(p_ra);
    std::cout << " p_ra = " << p_ra << std::endl
    << " p_pi = " << p_pi << std::endl
    << " p_dg = " << p_dg << std::endl
    << " -> bs::cos(p_ra) = " << bs::cos(p_ra) << std::endl
    << " -> bs::cospi(p_pi) = " << bs::cospi(p_pi) << std::endl
    << " -> bs::cosd(p_dg) = " << bs::cosd(p_dg) << std::endl;
    return 0;
    }
    Possible output:
    p_ra = (-3.14159, -2.35619, -1.5708, -0.785398, 0, 0.785398, 1.5708, 2.35619)
    p_pi = (-1, -0.75, -0.5, -0.25, 0, 0.25, 0.5, 0.75)
    p_dg = (-180, -135, -90, -45, 0, 45, 90, 135)
    -> bs::cos(p_ra) = (-1, -0.707107, -4.37114e-08, 0.707107, 1, 0.707107, -4.37114e-08, -0.707107)
    -> bs::cospi(p_pi) = (-1, -0.707107, -0, 0.707107, 1, 0.707107, -0, -0.707107)
    -> bs::cosd(p_dg) = (-1, -0.707107, -0, 0.707107, 1, 0.707107, -0, -0.707107)

    All direct and inverse functions (except cardinal ones) are provided in three flavours using respectively radian, degrees and \(\pi\) multiples.

    Be aware that their uses are not equivalent (see the above results).

    To give another example: sinpi(1.0) is not equal to sin(Pi<double>()).

    • Pi<double>() is only an approximation of \(\pi\) to 0.5 ulp and the reduction of the angle Pi<double>() between \(-\pi/4\) and \(\pi/4\) will lead to a non zero value and sin will return this small value (namely 1.22464679914735e-16) as \(\sin x \sim x\) for small values.
    • 1.0 is an exact double value and the reduction of the angle 1.0 between \(-0.25\) and \(0.25\) will lead to an exact 0.0 the sinus of which is exactly 0.0.
    • However the two results are correct to 0.5 ulp.
    • All this DOES NOT IMPLY that trigonometric functors taking degrees or \(\pi\) multiples as inputs are more accurate than those taking radians. The value computed by sin(Pi<double>()) is the correct value up to 0.5ulp of the sine of the input (which is not equal to \(\pi\)).

    'pi' (resp. 'd') suffix indicates that the return value is in \(\pi\) multiples (resp. degrees).

    Example:
    #include <boost/simd/trigonometric.hpp>
    #include <boost/simd/pack.hpp>
    #include <boost/simd/function/enumerate.hpp>
    #include <iostream>
    namespace bs = boost::simd;
    using pack_ft = bs::pack <float, 8>;
    int main()
    {
    pack_ft p = bs::enumerate<pack_ft>(-1.0f, 0.25f);
    std::cout << " p = " << p << std::endl
    << " -> bs::acos(p) = " << bs::acos(p) << std::endl
    << " -> bs::acospi(p) = " << bs::acospi(p) << std::endl
    << " -> bs::acosd(p) = " << bs::acosd(p) << std::endl;
    return 0;
    }
    Possible output:
    p = (-1, -0.75, -0.5, -0.25, 0, 0.25, 0.5, 0.75)
    -> bs::acos(p) = (3.14159, 2.41886, 2.0944, 1.82348, 1.5708, 1.31812, 1.0472, 0.722734)
    -> bs::acospi(p) = (3.14159, 2.41886, 2.0944, 1.82348, 1.5708, 1.31812, 1.0472, 0.722734)
    -> bs::acosd(p) = (3.14159, 2.41886, 2.0944, 1.82348, 1.5708, 1.31812, 1.0472, 0.722734)

    inrad converts degree in radian

  • Reduction utilities

    The algorithms used here to compute sine or cosine functions have two main steps

    • reduction of the angle to the \([-\pi/4, \pi/4]\) interval (if expressed in radian) and calculation of the right quadrant.
    • evaluation of an approximation polynomial on the reduced interval.

    The reduction of the angle when no information is known on the angle magnitude can be very costly.

    there actually are 4 reduction routines protocol that respectively are sufficient for small_, medium_ and big_ angle values to have within cover test one ulp of difference with the according crlibm (correctly rounded math library) result.

    In fact the standard reduction algorithm used is roughly (M is \(2^6\pi\) for float and \(2^{18}\pi\) for double)

    • if ( \(|x|\) is in \([0, \pi/4]\)) do nothing
    • else if ( \(|x|\) is in \([0, \pi/2]\)) use rem_pio2_straight
    • else if ( \(|x|\) is in \([0, 20\pi]\)) use rem_pio2_cephes
    • else if ( \(|x|\) is in \([0, M\pi]\)) use rem_pio2_medium
    • else use rem_pio2

    There also exists rem_2pi that only computes the reduction of the radian angle between \([-\pi, \pi]\)
